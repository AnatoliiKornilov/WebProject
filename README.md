# DevPortfolio

DevPortofio - интерактивное портфолио программиста. Проект реализовал Корнилов Анатолий Александрович. 
На этом сайте можно выкладывать всю необходимую информацию о себе, например, ФИО, дату рождения, стек технологий, 
опыт работы, образование, ссылки на GitHub, GitLab. Основной фичей проекта является возможность выкладывать свои 
проекты по одному с подробной информацией о них, такой как название, описание, стек технологий, роль человека в проекте, изображение, ссылку на задеплоенное приложение, ссылку на исходный код. На этом сайте можно смотреть также чужие портфолио, то есть можно скинуть работодателю ссылку на публичный профиль на этом сайте и он получит всю подробную информацию про программиста в удобном красивом виде.

## Сценарии использования
Вы можете пользоваться DevPortfolio без создания аккаунта. Но тогда вы сможете только смотреть чужие портфолио и искать их, 
пользуясь поисковой строкой на соответствующей странице.

Все остальные действия доступны только авторизованным пользователям. Для этого вам необходимо зарегистрироваться в системе.
### ВАЖНО!!!
Вводите свою настоящую почту. После регистрации вам придёт письмо с подтверждением почты. Только после подтверждения почты вы сможете полноценно пользоваться аккаунтом. 
Авторизованный пользователь может редактировать свой профиль, то есть добавлять данные о себе и своих проектах. После авторизации 
профиль пользователя будет виден всем.

## Для запуска БД
Запустить локально на своём устройстве Docker.
Из корня проекта прописать:
npx supabase start -x vector

## Для запуска приложения
Сначала запустить бд в отдельной консоли
Из папки frontend:
npm run dev

## Для запуска тестов
Из папки frontend:
npm test

## Для запуска E2E-тестов
В одной консоли запустить приложение, в другой из папки frontend:
npm run test:e2e

## Структура проекта
В корне есть директории frontend и supabase. В папке supabase почти ничего нет, кроме sql-запросов для создания таблиц в бд.
В папке frontend из основного есть папки tests для настройки запуска тестов и src для основного кода.
В папке src из основного - папки __tests__ с кодом тестов, components - разные переиспользуемые компоненты проекта, 
lib - для связи с БД и типами того, что в ней лежит, pages - компоненты страниц, store - для работы с redux toolkit и 
запросов к бд, types - типы для всего проекта. В App.tsx отрисовываются основные компоненты и настраивается react router.
